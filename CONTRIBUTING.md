## 日本語

プロジェクトへ興味を持っていただき、ありがとうございます。
円滑な開発とレビューのために、以下のガイドラインに沿ってご参加ください。

### 参加する前の確認
- 既に同じ内容のIssueやPull Request（PR）が作成されていないか、事前に必ず検索して確認してください。

### Issue（イシュー）について
- バグ報告や新機能の提案は、用意されている専用のテンプレートに沿って記述してください。
- 1つのIssueには、関連する1つのテーマのみを記載してください。

### Pull Request（プルリクエスト）について
- PRを作成する際は、必ず関連するIssueとの紐付けを行ってください。
- レビューしやすい規模を保つため、1つのPRは「1つの目的（トピック）」に集中させてください。関連する変更点（項目）は、目安として合計で最大5個までとします。変更点が多すぎると差分が膨れ、レビューする側の負荷が跳ね上がってしまうため、それを超える規模の変更はPRを複数に分割して提出してください。
- テンプレートの各項目（What/Why/How/Reason）は、可能な限り省略せずに記載してください。

## English

Thank you for your interest in this project!
To ensure a smooth development and review process, please follow the guidelines below.

### Before Contributing
- Please search existing Issues and Pull Requests (PRs) to ensure your topic has not already been covered.

### Issues
- Please use the provided templates for bug reports or feature requests.
- Each Issue should focus on a single topic or problem.

### Pull Requests
- All PRs must be linked to a relevant Issue.
- To maintain a reviewable size, a single PR should focus on "one specific purpose (topic)." As a guideline, please limit the related changes to a maximum of 5 items per PR. A large number of changes bloats the diff and significantly increases the reviewer's burden, so if your changes exceed this, please split them into multiple PRs.
- Please fill out all sections (What/Why/How/Reason) in the template as accurately as possible.
